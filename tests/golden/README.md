# Golden Test Data

This directory contains reference data used by Marmot's backend and graph tests to validate correctness against known-good outputs. Golden data comes from two sources: llama.cpp (for quantization and embedding operations) and NumPy (for floating-point activations and graph operations).

## Data Sources

### llama.cpp (quantization, embeddings, matmul, vec_dot)

Golden data for quantization round-trips, embedding gather, matrix multiplication, and vector dot products is generated by compiling a C program (`gen_golden_from_llama.c`) against ggml from llama.cpp. This ensures Marmot's quantized operations produce bit-identical results to the reference implementation.

**Generated header files** (written to `tests/backend/`):

| File | Contents |
|------|----------|
| `golden_quant_llama.h` | Quantization/dequantization round-trip reference values for Q4_0 through Q8_K |
| `golden_float_ops_llama.h` | Float operation reference values |
| `golden_matmul_llama.h` | Matrix multiplication reference values |
| `golden_vec_dot_llama.h` | Vector dot product reference values for each quantization scheme |

**Generated text fixtures** (written to `tests/golden/`):

| File | Contents |
|------|----------|
| `embedding_q4_0.txt` | Embedding gather output for Q4_0 quantized weights |
| `embedding_q4_1.txt` | Embedding gather output for Q4_1 quantized weights |
| `embedding_q5_0.txt` | Embedding gather output for Q5_0 quantized weights |
| `embedding_q5_1.txt` | Embedding gather output for Q5_1 quantized weights |
| `embedding_q8_0.txt` | Embedding gather output for Q8_0 quantized weights |
| `embedding_q8_1.txt` | Embedding gather output for Q8_1 quantized weights |
| `embedding_q4_0_ragged.txt` | Embedding gather output for non-block-aligned dimensions |

Each `embedding_*.txt` file begins with a comment that includes the llama.cpp commit SHA used to generate the data. Regenerate these files when updating ggml/llama.cpp to keep fixtures in sync.

### NumPy (unary activations, graph operations)

Floating-point activations and graph-level smoke tests are generated from NumPy to keep the workflow lightweight and reproducible without external model files.

**Unary activation goldens** (written to `tests/backend/`):

| File | Contents |
|------|----------|
| `golden_unary_numpy.h` | Float32 and float64 reference values for all unary activations (ReLU, GELU, SiLU, sigmoid, tanh, etc.) plus scalar/vector fused-bias variants |

**Graph matmul goldens** (written to `tests/graph/`):

| File | Contents |
|------|----------|
| `golden_graph_numpy.h` | Deterministic matmul fixtures for single-stage and two-stage pipeline graph tests |

**General backend goldens** (written to `tests/backend/`):

| File | Contents |
|------|----------|
| `golden_data.h` | Additional backend test reference data generated by `generate_backend_golden.py` |

## Generation Scripts

### llama.cpp goldens

```bash
cd tests/golden
./generate_from_llama.sh
```

This script:
1. Clones llama.cpp (shallow clone, if not already present).
2. Builds ggml as a static library.
3. Compiles `gen_golden_from_llama.c` against both ggml and Marmot.
4. Runs the generator, which produces all header files and text fixtures.

Prerequisites: CMake, a C++ compiler, and a built Marmot library (`ninja -C build`).

### NumPy unary activation goldens

```bash
uv run python tests/golden/generate_unary_numpy.py
```

Produces `tests/backend/golden_unary_numpy.h`.

### NumPy graph matmul goldens

```bash
uv run python tests/golden/generate_graph_numpy.py
```

Produces `tests/graph/golden_graph_numpy.h`. The generator uses NumPy when available and falls back to a pure-Python implementation for minimal environments.

### General backend goldens

```bash
uv run python tests/golden/generate_backend_golden.py
```

Produces `tests/backend/golden_data.h`.

## When to Regenerate

Regenerate golden data when:

- **Updating llama.cpp or ggml**: Run `./generate_from_llama.sh` to capture any changes in quantization behavior.
- **Adding new quantization schemes**: Update `gen_golden_from_llama.c` to include the new scheme, then regenerate.
- **Adding new unary activations**: Update `generate_unary_numpy.py` with the new activation function, then regenerate.
- **Changing graph test shapes or operations**: Update `generate_graph_numpy.py`, then regenerate.

After regeneration, run `make test` to verify all tests still pass.

## Adding a New Golden Test

1. **Choose the data source.** Use llama.cpp if the operation involves quantized types or must match ggml exactly. Use NumPy for floating-point math operations.

2. **Update the generator script.** Add the new test case to the appropriate generator (`gen_golden_from_llama.c`, `generate_unary_numpy.py`, `generate_graph_numpy.py`, or `generate_backend_golden.py`).

3. **Regenerate.** Run the appropriate generation command.

4. **Write the test.** In the test file under `tests/backend/` or `tests/graph/`, include the generated header and compare Marmot's output against the golden values at an appropriate tolerance.

5. **Register the test.** Add the test to `tests/meson.build` if it is a new file.
