# Operation Signature Reference

Header: `include/marmot/graph/op_signature.h`

The `marmot_op_signature_t` structure is the primary mechanism for describing kernel requirements to the dispatch system. It captures the operation type, data types, memory layout constraints, quantization parameters, epilogue configuration, and operation-specific dimensions.

---

## Structure Definition

```c
typedef struct {
    marmot_op_id_t          op_id;
    marmot_profile_id_t     profile_id;
    marmot_matmul_layout_t  matmul_layout;

    marmot_dtype_t          input_dtype;
    marmot_dtype_t          weight_dtype;
    marmot_dtype_t          output_dtype;
    marmot_dtype_t          accum_dtype;

    marmot_qscheme_id_t     qscheme_id;
    marmot_quant_block_t    quant_block;

    marmot_weight_layout_t  weight_layout;
    marmot_stride_mode_t    stride_mode;
    uint32_t                epilogue_flags;
    marmot_device_unary_op_t activation;
    uint32_t                variant_flags;

    marmot_op_signature_dims_t dims;
} marmot_op_signature_t;
```

---

## Core Fields

### `op_id` (`marmot_op_id_t`)

Identifies the operation to perform. Operation IDs are generated by codegen from kernel `.def` files and defined in `include/marmot/traits_ids.gen.h`.

Common categories:
- **Matmul**: `MARMOT_OP_MATMUL`, `MARMOT_OP_MATMUL_BIAS`, `MARMOT_OP_MATMUL_BIAS_RELU`, `MARMOT_OP_MATMUL_BIAS_GELU`, `MARMOT_OP_MATMUL_BIAS_SILU`
- **Elementwise binary**: `MARMOT_OP_ADD`, `MARMOT_OP_MUL`, `MARMOT_OP_SUB`, `MARMOT_OP_DIV`
- **Elementwise unary**: `MARMOT_OP_RELU`, `MARMOT_OP_GELU`, `MARMOT_OP_SILU`, `MARMOT_OP_SIGMOID`
- **Fused elementwise**: `MARMOT_OP_ADD_RELU`, `MARMOT_OP_ADD_GELU`, `MARMOT_OP_ADD_SILU`, `MARMOT_OP_FMA`
- **Reduction**: `MARMOT_OP_REDUCTION_SUM`, `MARMOT_OP_REDUCTION_MAX`, `MARMOT_OP_REDUCTION_MEAN`
- **Normalization**: `MARMOT_OP_LAYERNORM`, `MARMOT_OP_RMS_NORM`, `MARMOT_OP_SOFTMAX`
- **Attention**: `MARMOT_OP_PAGED_ATTENTION`
- **Tensor manipulation**: `MARMOT_OP_RESHAPE`, `MARMOT_OP_TRANSPOSE`, `MARMOT_OP_CONCAT`

### `profile_id` (`marmot_profile_id_t`)

Specifies the implementation profile for CPU variant selection. Defined in `include/marmot/traits_ids.gen.h`.

| Value | Description |
|-------|-------------|
| `MARMOT_PROFILE_INVALID` | No specific profile (default; system auto-selects) |
| `MARMOT_PROFILE_ACCELERATE` | Apple Accelerate framework |
| `MARMOT_PROFILE_NEON` | ARM NEON SIMD |
| `MARMOT_PROFILE_AVX2` | Intel AVX2 SIMD |
| `MARMOT_PROFILE_AVX512` | Intel AVX-512 SIMD |
| `MARMOT_PROFILE_SCALAR` | Scalar fallback |

The kernel selection system automatically picks the best available profile based on runtime CPU capabilities.

### `matmul_layout` (`marmot_matmul_layout_t`)

For matmul operations, specifies the transpose configuration. Defined in `include/marmot/matmul_types.h`.

| Value | Description | Formula |
|-------|-------------|---------|
| `MARMOT_MATMUL_LAYOUT_NN` | Both operands non-transposed | C = A * B |
| `MARMOT_MATMUL_LAYOUT_NT` | B transposed | C = A * B^T |
| `MARMOT_MATMUL_LAYOUT_TN` | A transposed | C = A^T * B |
| `MARMOT_MATMUL_LAYOUT_TT` | Both transposed | C = A^T * B^T |
| `MARMOT_MATMUL_LAYOUT_INVALID` | Not applicable (non-matmul ops) | -- |

NT is the most common layout for LLM inference because weights are stored transposed for efficient memory access.

## Data Type Fields

### `input_dtype` (`marmot_dtype_t`)

Data type of the primary input tensor. Defined in `include/marmot/types.h`.

| Value | Description | Size |
|-------|-------------|------|
| `MARMOT_DTYPE_FLOAT32` | 32-bit float | 4 bytes |
| `MARMOT_DTYPE_FLOAT16` | 16-bit float (IEEE 754) | 2 bytes |
| `MARMOT_DTYPE_BFLOAT16` | Brain float 16 | 2 bytes |
| `MARMOT_DTYPE_FLOAT64` | 64-bit float | 8 bytes |
| `MARMOT_DTYPE_FLOAT8_E4M3` | FP8 (4-bit exponent, 3-bit mantissa) | 1 byte |
| `MARMOT_DTYPE_FLOAT8_E5M2` | FP8 (5-bit exponent, 2-bit mantissa) | 1 byte |
| `MARMOT_DTYPE_INT8` | Signed 8-bit integer | 1 byte |
| `MARMOT_DTYPE_INT16` | Signed 16-bit integer | 2 bytes |
| `MARMOT_DTYPE_INT32` | Signed 32-bit integer | 4 bytes |
| `MARMOT_DTYPE_INT64` | Signed 64-bit integer | 8 bytes |
| `MARMOT_DTYPE_UINT8` | Unsigned 8-bit integer | 1 byte |
| `MARMOT_DTYPE_UINT16` | Unsigned 16-bit integer | 2 bytes |
| `MARMOT_DTYPE_UINT32` | Unsigned 32-bit integer | 4 bytes |
| `MARMOT_DTYPE_UINT64` | Unsigned 64-bit integer | 8 bytes |

### `weight_dtype` (`marmot_dtype_t`)

Data type of the weight or second operand tensor. For non-quantized operations, typically matches `input_dtype`. For quantized operations, set to the packed quantized type.

### `output_dtype` (`marmot_dtype_t`)

Data type of the output tensor. Usually matches `input_dtype` but may differ for comparison operations (boolean output), mixed-precision operations, or type conversion operations.

### `accum_dtype` (`marmot_dtype_t`)

Accumulator data type for reductions and matmul. Controls precision during intermediate computation.

| Use Case | Typical Value |
|----------|---------------|
| Float operations | `MARMOT_DTYPE_FLOAT32` |
| Double operations | `MARMOT_DTYPE_FLOAT64` |
| Integer operations | Same as input dtype |
| Layout ops (reshape, etc.) | `MARMOT_DTYPE_COUNT` (ANY) |

## Quantization Fields

### `qscheme_id` (`marmot_qscheme_id_t`)

Identifies the quantization scheme for quantized weight operations. Defined in `include/marmot/traits_ids.gen.h`.

| Value | Description |
|-------|-------------|
| `MARMOT_QSCHEME_NONE` | No quantization (dense weights) |
| `MARMOT_QSCHEME_Q4_0` | 4-bit, no zero-point |
| `MARMOT_QSCHEME_Q4_1` | 4-bit with zero-point |
| `MARMOT_QSCHEME_Q5_0` | 5-bit, no zero-point |
| `MARMOT_QSCHEME_Q5_1` | 5-bit with zero-point |
| `MARMOT_QSCHEME_Q8_0` | 8-bit, no zero-point |
| `MARMOT_QSCHEME_Q8_1` | 8-bit with zero-point |
| `MARMOT_QSCHEME_Q2_K` | K-quant 2-bit |
| `MARMOT_QSCHEME_Q3_K` | K-quant 3-bit |
| `MARMOT_QSCHEME_Q4_K` | K-quant 4-bit |
| `MARMOT_QSCHEME_Q5_K` | K-quant 5-bit |
| `MARMOT_QSCHEME_Q6_K` | K-quant 6-bit |
| `MARMOT_QSCHEME_Q8_K` | K-quant 8-bit |

### `quant_block` (`marmot_quant_block_t`)

Describes the quantization block structure:

```c
typedef struct {
    uint32_t block_size;           // Elements per quantization block (typically 32 or 64)
    uint32_t group_size;           // Blocks per group sharing metadata
    marmot_dtype_t scale_dtype;    // Scale factor data type (usually FLOAT16)
    marmot_dtype_t zero_point_dtype; // Zero-point data type, or MARMOT_DTYPE_COUNT if none
} marmot_quant_block_t;
```

## Memory Layout Fields

### `weight_layout` (`marmot_weight_layout_t`)

Specifies how QKV projection weights are organized in memory.

| Value | Description |
|-------|-------------|
| `MARMOT_WEIGHT_LAYOUT_INVALID` | Not applicable (default for most ops) |
| `MARMOT_WEIGHT_LAYOUT_SEPARATE` | Q, K, V weights in separate tensors |
| `MARMOT_WEIGHT_LAYOUT_PACKED_3MK` | Q, K, V packed as [3, M, K] tensor |

### `stride_mode` (`marmot_stride_mode_t`)

Declares the memory layout requirements for the kernel.

| Value | Description | Use Case |
|-------|-------------|----------|
| `MARMOT_STRIDE_MODE_ANY` | No layout constraint | Layout-agnostic ops |
| `MARMOT_STRIDE_MODE_CONTIGUOUS` | Fully contiguous in memory | SIMD-optimized kernels |
| `MARMOT_STRIDE_MODE_ROW_STRIDED` | Contiguous within rows | Row-major batch ops |
| `MARMOT_STRIDE_MODE_STRIDED` | Arbitrary strides allowed | Flexible but slower |

If a tensor does not match the kernel's stride mode, the system tries to find an alternative kernel with looser requirements, or falls back to making the tensor contiguous before dispatch.

## Epilogue and Fusion Fields

### `epilogue_flags` (`uint32_t`)

Bitmask of post-operation transformations. Flags can be combined with bitwise OR.

| Flag | Value | Description |
|------|-------|-------------|
| `MARMOT_EPILOGUE_NONE` | `0` | No epilogue |
| `MARMOT_EPILOGUE_BIAS` | `1 << 0` | Add bias vector after main op |
| `MARMOT_EPILOGUE_ACTIVATION` | `1 << 1` | Apply activation function |
| `MARMOT_EPILOGUE_RESIDUAL` | `1 << 2` | Add residual connection |
| `MARMOT_EPILOGUE_ROPE` | `1 << 3` | Apply rotary position embedding |

### `activation` (`marmot_device_unary_op_t`)

When `MARMOT_EPILOGUE_ACTIVATION` is set, specifies which activation to apply. Values from `include/marmot/ops_types.h`:

| Value | Function |
|-------|----------|
| `MARMOT_DEVICE_UNARY_IDENTITY` | f(x) = x (no-op) |
| `MARMOT_DEVICE_UNARY_RELU` | f(x) = max(0, x) |
| `MARMOT_DEVICE_UNARY_GELU` | Gaussian Error Linear Unit |
| `MARMOT_DEVICE_UNARY_GELU_TANH` | GELU with tanh approximation |
| `MARMOT_DEVICE_UNARY_SILU` | f(x) = x * sigmoid(x) |
| `MARMOT_DEVICE_UNARY_SIGMOID` | f(x) = 1 / (1 + exp(-x)) |
| `MARMOT_DEVICE_UNARY_TANH` | Hyperbolic tangent |

### `variant_flags` (`uint32_t`)

Kernel variant selection bitmask using internal `MARMOT_FUSION_*` flags (defined in `src/core/dispatch/fusion_flags.h`). Used for selecting kernels that incorporate extra semantics into a single op (e.g., normalization variants that also add a residual input).

If kernel selection fails with non-zero `variant_flags`, the system retries with `variant_flags = 0` as a fallback.

## Dimension Union (`dims`)

The `dims` field is a union carrying operation-specific dimension information. Only one variant is valid at a time, determined by `op_id`.

### `marmot_matmul_dims_t`

For matmul operations (`MARMOT_OP_MATMUL`, `MARMOT_OP_MATMUL_BIAS`, `MARMOT_OP_MATMUL_BIAS_*`):

```c
typedef struct {
    size_t N;  // Rows (batch dimension)
    size_t K;  // Reduction dimension
    size_t M;  // Columns (output features)
} marmot_matmul_dims_t;
```

### `marmot_layernorm_dims_t`

For normalization operations (`MARMOT_OP_LAYERNORM`, `MARMOT_OP_RMS_NORM`):

```c
typedef struct {
    uint32_t n;  // Batch size (outer dimension)
    uint32_t d;  // Feature dimension (normalized axis)
} marmot_layernorm_dims_t;
```

### `marmot_rope_dims_t`

For rotary position embedding (`MARMOT_OP_ROPE`):

```c
typedef struct {
    uint32_t seq_len;  // Sequence length
    uint32_t n_rot;    // Number of rotary dimensions
} marmot_rope_dims_t;
```

### `marmot_elementwise_dims_t`

For all elementwise binary and unary operations:

```c
typedef struct {
    uint32_t n_elems;  // Total number of elements
} marmot_elementwise_dims_t;
```

### `marmot_softmax_dims_t`

For softmax (`MARMOT_OP_SOFTMAX`):

```c
typedef struct {
    uint32_t axis_size;     // Size of the softmax axis
    uint32_t inner_stride;  // Stride between elements on the axis
    uint32_t outer_size;    // Number of independent softmax instances
    uint32_t row_count;     // Rows to process
} marmot_softmax_dims_t;
```

## Usage Examples

### Dense Matmul (C = A * B^T)

```c
marmot_op_signature_t sig = {
    .op_id = MARMOT_OP_MATMUL,
    .matmul_layout = MARMOT_MATMUL_LAYOUT_NT,
    .input_dtype = MARMOT_DTYPE_FLOAT16,
    .weight_dtype = MARMOT_DTYPE_FLOAT16,
    .output_dtype = MARMOT_DTYPE_FLOAT16,
    .accum_dtype = MARMOT_DTYPE_FLOAT32,
    .stride_mode = MARMOT_STRIDE_MODE_CONTIGUOUS,
    .dims.matmul = { .N = 128, .M = 512, .K = 256 },
};
```

### Quantized Matmul (Q4_K)

```c
marmot_op_signature_t sig = {
    .op_id = MARMOT_OP_MATMUL,
    .matmul_layout = MARMOT_MATMUL_LAYOUT_NT,
    .input_dtype = MARMOT_DTYPE_FLOAT16,
    .output_dtype = MARMOT_DTYPE_FLOAT16,
    .accum_dtype = MARMOT_DTYPE_FLOAT32,
    .qscheme_id = MARMOT_QSCHEME_Q4_K,
    .quant_block = {
        .block_size = 32,
        .group_size = 8,
        .scale_dtype = MARMOT_DTYPE_FLOAT16,
        .zero_point_dtype = MARMOT_DTYPE_COUNT,  // None
    },
    .dims.matmul = { .N = 1, .M = 4096, .K = 4096 },
};
```

### Fused Matmul + Bias + ReLU

```c
marmot_op_signature_t sig = {
    .op_id = MARMOT_OP_MATMUL_BIAS_RELU,
    .matmul_layout = MARMOT_MATMUL_LAYOUT_NT,
    .input_dtype = MARMOT_DTYPE_FLOAT32,
    .weight_dtype = MARMOT_DTYPE_FLOAT32,
    .output_dtype = MARMOT_DTYPE_FLOAT32,
    .accum_dtype = MARMOT_DTYPE_FLOAT32,
    .epilogue_flags = MARMOT_EPILOGUE_BIAS | MARMOT_EPILOGUE_ACTIVATION,
    .activation = MARMOT_DEVICE_UNARY_RELU,
    .dims.matmul = { .N = 64, .M = 256, .K = 128 },
};
```

### Elementwise Addition

```c
marmot_op_signature_t sig = {
    .op_id = MARMOT_OP_ADD,
    .input_dtype = MARMOT_DTYPE_FLOAT32,
    .output_dtype = MARMOT_DTYPE_FLOAT32,
    .accum_dtype = MARMOT_DTYPE_FLOAT32,
    .stride_mode = MARMOT_STRIDE_MODE_CONTIGUOUS,
    .dims.elementwise = { .n_elems = 1024 },
};
```

### LayerNorm

```c
marmot_op_signature_t sig = {
    .op_id = MARMOT_OP_LAYERNORM,
    .input_dtype = MARMOT_DTYPE_FLOAT16,
    .output_dtype = MARMOT_DTYPE_FLOAT16,
    .accum_dtype = MARMOT_DTYPE_FLOAT32,
    .dims.layernorm = { .n = 32, .d = 768 },
};
```
